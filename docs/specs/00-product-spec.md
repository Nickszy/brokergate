# Product Spec

## 定位

OpenBroker 是开源、自部署、多券商交易网关。它提供统一 API、基础 Web 配置、人工确认下单、安全审计，以及后续 MCP / Claude / OpenClaw 集成能力。

它不是自动荐股或自动炒股产品。产品红线是：AI 可以查询账户、生成交易草稿和解释风险，但不能绕过用户确认直接执行交易。

## 目标用户

- 有券商 API 使用经验的个人投资者。
- 需要统一管理 Tiger、Futu、Longbridge 的港美股投资者。
- 量化/开发者用户，希望在自有服务器上运行交易基础设施。
- 希望把 AI 分析结果接到账户查询、交易草稿、复盘工作流的人。

## 首版价值

- 用户密钥留在自己的服务器上。
- 通过统一模型调用券商查询和下单能力。
- 每笔订单先成为草稿，再由用户确认。
- 所有敏感操作都有审计日志。
- Tiger 首发，后续扩展到 Futu 和 Longbridge。

## 非目标

- 不提供投资建议。
- 不承诺收益。
- 不做全自动交易机器人。
- 不托管用户券商密钥。
- 不在首版实现多用户团队权限和 Pro 分析。

## MVP 1 范围

- Tiger 账号连接配置。
- 查询账户、现金、持仓。
- 创建订单草稿。
- 用户二次确认。
- 提交订单。
- 订单和操作审计。
- Docker 部署。

## 成功指标

- 开发者能在 30 分钟内完成本地部署。
- 用户能用 Tiger paper/sandbox 模式完成查询和确认下单。
- 订单确认流程没有任何绕过路径。
- 至少收集 10 个目标用户反馈，回答是否愿意自部署、代部署或购买账户分析能力。

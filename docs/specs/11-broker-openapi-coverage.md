# 券商开放接口能力覆盖矩阵

## 口径

本文按“数据类型 -> 二级分类 -> 券商能力”的方式横向比较长桥、老虎、富途开放接口，并标注 BrokerGate 当前覆盖情况。

状态定义：

| 状态 | 含义 |
| --- | --- |
| 已接入 | BrokerGate 已有公开 HTTP 接口或完整交易流程调用到对应券商能力 |
| 部分接入 | 适配器代码已经调用了券商能力，但只覆盖子集，或还没有暴露成公开 HTTP 接口 |
| 规划占位 | BrokerGate 有统一接口约束或状态占位，但没有真实适配器实现 |
| 未接入 | 当前没有相关代码路径 |

BrokerGate 当前目标是统一交易入口，不是完整复制券商开放平台。现阶段最小闭环是：账户快照、购买力风控、订单草稿、人工确认、券商下单和审计。

## 当前最值得补的能力

| 优先级 | 能力 | 原因 |
| --- | --- | --- |
| 1 | 富途适配器方案 | 老虎和长桥交易管理能力已经先接上；接富途前必须确定 OpenD 部署、交易解锁、账户绑定和重连策略 |
| 2 | 长桥订单费用 | 长桥主交易目录没有单列费用接口，需要确认 SDK 是否有替代字段，不能伪装成功 |
| 3 | 推送式订单状态同步 | 当前老虎和长桥是拉取式同步，还没有接交易推送 |
| 4 | 改单预览接口 | 当前改单提交会校验确认文本和重新风控，但还没有独立的改单预览/确认文本生成接口 |
| 5 | 行情订阅 | 当前只接入老虎和长桥的拉取式报价快照，没有接 WebSocket 订阅；拉取式行情已支持只读 fallback |
| 6 | 更广泛市场数据 | K 线、期权、基本面、新闻等能力仍不应排在交易安全闭环之前 |

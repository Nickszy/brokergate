# BrokerGate 产品定位

## 一句话

BrokerGate 是一个面向中文用户的自部署多券商账户与交易网关。

它通过券商官方 OpenAPI，把用户本人已开通的海外券商账户接回自己的控制台，让用户可以在一个地方：

- 看账户：资产、现金、持仓、订单、成交；
- 做交易：生成订单草稿、经过风控检查、由用户人工确认后提交到券商。

BrokerGate 不是券商，不托管用户资金，不提供投资建议，不自动替用户交易。

---

## 目标用户

第一阶段优先服务中文用户，而不是海外开发者社区。

更具体地说，目标用户包括：

1. 已经拥有海外券商账户的中文用户；
2. 使用老虎、长桥、富途、IBKR 等多个券商的人；
3. 希望统一查看账户、持仓、订单和成交记录的人；
4. 希望用模拟账户先跑通券商官方 OpenAPI 的用户；
5. 想把账户和交易能力接入个人脚本、看板或 AI Agent 的用户；
6. 不想把券商密钥交给中心化平台、愿意自部署的人。

---

## 核心用户痛点

### 1. 账户分散

海外券商账户越来越多，但资产、现金、持仓、订单和成交记录分散在不同 App 或接口里。

用户需要一个自己的账户控制台。

### 2. 交易入口分散

用户可能同时有多个券商账户，但不同券商的下单界面、API、字段和订单状态都不一致。

用户需要一个统一的交易入口。

### 3. API 门槛高

老虎、长桥、富途、IBKR 都有自己的 OpenAPI，但普通用户或轻量开发者不一定愿意从零学习每一家的 SDK。

BrokerGate 用统一接口降低接入成本。

### 4. AI Agent 缺少安全的账户能力

很多用户想让 AI 读取账户、总结持仓、生成交易草稿，但不希望 AI 绕过人工确认直接下单。

BrokerGate 提供可审计、可确认、可关闭的账户与交易能力。

---

## 产品双核心

BrokerGate 不是只读资产看板，也不是纯交易下单工具。

它的双核心是：

```text
看账户 + 做交易
```

### 看账户

- 总资产
- 现金余额
- 购买力
- 持仓明细
- 订单记录
- 成交记录
- 券商连接状态
- 多券商统一视图

### 做交易

- 创建订单草稿
- 刷新券商账户快照
- 执行风控检查
- 用户人工确认
- 提交到券商官方 SDK / OpenAPI
- 记录审计日志

---

## 第一阶段产品边界

第一阶段不追求完整复制券商 App，也不做复杂行情和社区功能。

优先做：

1. 本地 paper fallback；
2. 真实券商模拟账户接入；
3. 账户、持仓、订单、成交统一接口；
4. 订单草稿与人工确认链路；
5. Docker / Render 自部署；
6. 中文 README、文档和部署说明。

暂时不做：

1. 自动交易；
2. AI 推荐买卖；
3. 跟单或策略市场；
4. 托管用户资金；
5. 中心化保存用户券商密钥；
6. 绕过券商或监管限制的宣传。

---

## 安全与合规表达

对外表达时，必须长期坚持以下几句话：

- BrokerGate 使用券商官方 OpenAPI；
- BrokerGate 不做屏幕抓取，不模拟点击券商 App；
- BrokerGate 不提供证券经纪、投资咨询或资产管理服务；
- BrokerGate 不托管用户资金；
- BrokerGate 默认模拟账户优先；
- BrokerGate 真实交易必须由用户人工确认；
- 用户需要自行确认其券商账户、API 使用方式和交易行为符合所在地法律法规及券商协议。

## 营销主张

### 主标题方向

多个海外券商账户，终于可以放在一个地方看了。

### 副标题方向

一个页面看账户，一个入口做交易。BrokerGate 通过券商官方 OpenAPI，把资产、持仓、订单、成交和交易确认能力接回你的自部署控制台。

### 群里介绍版

我在做一个叫 BrokerGate 的开源项目。它不是券商，也不提供投资建议，而是一个面向中文用户的自部署多券商账户与交易网关。第一阶段支持通过官方 OpenAPI 连接老虎、长桥等券商，先跑模拟账户，再接真实账户；既能统一看资产、持仓、订单和成交，也能生成订单草稿，经风控和人工确认后提交到券商。

### 小红书 / 即刻版

我想做一个中文用户能用的海外券商账户控制台：

- 一个页面看清多个券商账户；
- 一个入口生成和确认交易；
- 默认先跑模拟账户；
- 密钥放在自己的本地或云服务器；
- 后续可以接 AI Agent 做持仓总结和订单草稿。

## 增长路径

### 第一步：GitHub + 中文社群

先吸引愿意折腾的第一批用户：

- 老虎 / 长桥 / 富途用户；
- 港美股投资者；
- AI 投资爱好者；
- 会用 Docker / VPS 的中文开发者；
- 想用券商 API 做自动化但不想从零接每家 SDK 的人。

### 第二步：降低部署门槛

提供：

- Docker Compose；
- Render 一键部署；
- Railway / Fly.io 后续支持；
- 中文环境变量说明；
- 模拟账户配置教程。

### 第三步：做出可展示的 Demo

即使用户暂时不填真实券商凭证，也能用本地 paper fallback 看到：

- 资产；
- 持仓；
- 订单草稿；
- 人工确认；
- 审计日志。

### 第四步：引导真实模拟账户接入

让用户先用券商模拟账户验证：

- 连接是否可用；
- 数据是否正确；
- 下单草稿和确认链路是否符合预期。

### 第五步：真实账户与交易确认

在用户充分理解风险后，再支持真实账户模式，并保持：

- 交易默认关闭；
- 人工确认默认开启；
- 风控规则默认启用；
- 操作日志可追溯。

---

## 不要怎么宣传

不建议使用以下表达：

- 绕过券商限制；
- 帮大陆用户继续交易；
- AI 自动炒股；
- 一键财富自由；
- 替你交易；
- 无需券商账户；
- 无需官方 API；
- 收益增强；
- 智能荐股。

这些表达会让项目偏离工具和基础设施定位，也会增加不必要的合规、信任和平台风险。

---

## 当前最重要的判断

BrokerGate 的短期价值不是做一个完整券商 App，而是先建立一个中文用户能理解、能自部署、能连接模拟账户、能同时看账户和确认交易的统一入口。

真正的增长抓手是：

```text
中文用户 + 海外券商账户 + 官方 OpenAPI + 自部署 + 看账户 + 做交易 + AI Agent
```

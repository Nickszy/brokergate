## 第五屏：模拟账户优先

### 标题

先跑模拟账户，再接真实账户

### 文案

BrokerGate 第一阶段推荐使用模拟账户或 paper 模式：

1. 不填券商凭证，使用本地 paper fallback 快速体验；
2. 填入券商官方模拟账户凭证，跑通真实 API 链路；
3. 确认账户、持仓、订单和交易确认流程稳定后，再谨慎切换真实账户；
4. 真实交易必须显式开启，并保持人工确认。

### 强调

默认配置：

```env
BROKERGATE_BROKER_MODE=paper
BROKERGATE_CONFIRMATION_REQUIRED=true
```

## 第十屏：FAQ

### BrokerGate 是券商吗？

不是。BrokerGate 是开源软件工具，不提供开户、清算、资金托管、投资咨询或资产管理服务。

### BrokerGate 会保存我的券商密钥吗？

优先部署在你自己的本地、VPS 或云账号中。BrokerGate 官方不需要接触你的券商密钥。

### 可以直接真实交易吗？

BrokerGate 默认是 paper / 模拟模式。真实交易需要显式设置 `BROKERGATE_BROKER_MODE=live-trade`，并建议始终保留人工确认。

### AI 可以自动帮我下单吗？

不建议。BrokerGate 的设计原则是：AI 可以读取账户、生成订单草稿，但不能绕过用户确认直接交易。

### 支持哪些券商？

当前项目已经包含老虎、长桥相关适配设计和能力，后续可扩展富途、IBKR 等券商。

### 这是不是绕过券商 App？

不是。BrokerGate 使用券商官方 OpenAPI，用户需要拥有本人券商账户和相应 API 凭证。
